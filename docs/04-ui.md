# GradingGuard AI — UI/UX Specification v0.1

## 1. Mục tiêu UI

UI của GradingGuard AI phải làm được 3 việc:

```text id="oxxdfo"
1. Giải thích vấn đề trong 10 giây.
2. Cho giám khảo thấy baseline AI grader bị thao túng điểm.
3. Cho giám khảo thấy GradingGuard AI phát hiện, làm sạch và bảo vệ điểm số.
```

Thông điệp chính trên UI:

```text id="xyghmc"
Protecting AI-based IELTS grading from prompt injection and score manipulation.
```

Không thiết kế như một app học IELTS thông thường. Thiết kế phải giống:

```text id="mz4u3h"
AI Security Dashboard
LLM Firewall Console
Trust & Safety Monitoring System
```

## 17. Demo-first UX rules

### 17.1. Always preload a good essay

Default essay must already be in textarea.

Do not make presenter type manually.

### 17.2. Vietnamese attack should be default

Default attack type:

```text id="2zuh9j"
direct_vietnamese
```

Reason:

```text id="d1eld8"
It clearly demonstrates why keyword-only English denylist is weak.
```

### 17.3. Buttons should follow demo order

Order:

```text id="fpf2s7"
1. Generate Attack
2. Run Baseline
3. Run Secure
4. Compare Results
```

### 17.4. Score cards should stay visible

Even after scroll, the score comparison must be easy to find.

### 17.5. Avoid too much text in main demo

Main page should show:

```text id="taw43b"
Risk
Action
Score difference
Removed span
```

Long explanations can go into expandable panels.

## 22. Final UI Story

Khi giám khảo nhìn UI, họ phải hiểu câu chuyện này:

```text id="ajvh17"
A normal essay receives Band 5.5.
A malicious instruction is injected into the essay.
The vulnerable baseline grader is manipulated to return Band 8.5.
GradingGuard AI detects the attack with high risk.
The malicious instruction is removed.
Secure grading returns Band 5.5.
The dashboard logs the event and proves score manipulation was prevented.
```

Đây là UI story quan trọng nhất.
