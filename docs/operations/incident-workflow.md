# Incident Workflow

High-risk or review-recommended decisions create persistent incidents.

Incident fields include source decision, correlation ID, severity, status, techniques, selected action, safe summary, restricted evidence reference and optimistic-lock version.

Phase 4 statuses are pilot-compatible with Phase 1:

- `awaiting_review`
- future: `open`, `triaged`, `investigating`, `resolved`, `dismissed`

All transitions that are implemented are audited.
